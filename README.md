# Fronter Log

A personal, local-first fronter-tracking and system-management app, backed by
an Obsidian vault. Built with Flutter.

All data is stored as plain files inside a `FronterLog` folder in your chosen
Obsidian vault:

```
<your-vault>/FronterLog/
  members/
    <member-id>.md      # YAML frontmatter (name, pronouns, role, color, avatar) + notes body
  avatars/
    <member-id>.jpg      # uploaded avatar images
  fronting_log.csv       # member_id, start, end, notes
```

Nothing is uploaded anywhere. The optional local API server (Settings tab)
just exposes read-only JSON endpoints on your own device/network so you can
later point a script or LLM at it.

## One-time setup

This project only ships the Dart/Flutter source (`lib/`, `pubspec.yaml`).
The platform-specific folders (`android/`, `ios/`, `linux/`, etc.) are
generated by the Flutter tool itself, so you'll create them once:

1. **Install Flutter**: https://docs.flutter.dev/get-started/install
   Run `flutter doctor` and make sure Android toolchain is green if you're
   targeting Android first.

2. **Unzip this project** somewhere, then from inside the folder run:

   ```bash
   flutter create . --project-name fronter_log --org com.yourname
   ```

   This fills in `android/`, plus `linux/`, `macos/`, `windows/` if you want
   them later. It will NOT overwrite `lib/` or `pubspec.yaml` since those
   already exist -- it only adds the missing platform scaffolding. If it
   prompts to overwrite a file like `.gitignore` or `analysis_options.yaml`,
   that's fine either way.

3. **Get packages**:

   ```bash
   flutter pub get
   ```

4. **Edit `android/app/src/main/AndroidManifest.xml`** and add these
   permissions inside the `<manifest>` tag (above `<application>`):

   ```xml
   <uses-permission android:name="android.permission.INTERNET"/>
   <uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>
   <uses-permission android:name="android.permission.CAMERA"/>
   <uses-permission android:name="android.permission.READ_MEDIA_IMAGES"/>
   ```

   `INTERNET` is needed for the local API server to bind/listen (no data
   leaves your network), `POST_NOTIFICATIONS` for the persistent current-
   fronter banner on Android 13+, and the last two for avatar uploads.

5. **Set the minimum SDK** (needed for `flutter_local_notifications` /
   `image_picker`) in `android/app/build.gradle`, inside `android { defaultConfig { ... } }`:

   ```gradle
   minSdkVersion 23
   ```

6. **Run it**:

   ```bash
   flutter run
   ```

   Pick your connected Android device/emulator when prompted. On first
   launch, go to **Settings** and choose your Obsidian vault folder (you'll
   need the Storage/"All files" permission granted, since it uses Android's
   native folder picker via `file_picker`).

## Known limitations (v0.1)

- The persistent "current fronter" notification is an *ongoing* notification,
  not a true foreground service -- Android may still kill it under aggressive
  battery optimization on some devices. Whitelisting the app from battery
  optimization in Android settings will make it much more reliable. A true
  foreground service (via `flutter_foreground_task`) is a natural upgrade
  later.
- The local API server has no authentication -- only enable it on networks
  you trust.
- Desktop (Windows/Linux/macOS) targets aren't wired up yet; the Dart code
  itself is cross-platform, but `file_picker`'s directory picker and
  notifications behave a bit differently per platform, so budget some testing
  time when you get to that stage.

## Where to go next

- Swap the ongoing notification for a real foreground service if persistence
  matters more than battery life.
- Add authentication (even a simple shared token) to the local API before
  exposing it beyond localhost.
- Build out desktop targets: `flutter create . ` again after adding
  `-platforms=windows,linux,macos` will add those folders too.
